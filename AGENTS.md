# CineLux Project Rules

These rules must be permanently followed for the CineLux project unless explicitly asked to change them.

## Movie Player Rules
- **Play Behavior**: When the user clicks the Play button, immediately pause the video before playback starts.
- **Instant Overlay**: Show the verification overlay instantly with no noticeable delay.
- **Paused State**: Keep the video paused while the verification overlay is visible.
- **Controls Lockdown**: Hide or disable all player controls while verification is in progress.
- **No Bypass**: Prevent users from bypassing the verification.
- **Auto-Close**: After successful verification, automatically close the verification overlay.
- **Auto-Resume**: Resume the video automatically from the exact position where it was paused.
- **Session Scope**: The verification should only be required once per browser session.
- **No Page Reload**: Do not reload the page.
- **Design Consistency**: Maintain the pristine player styling, layout, typography, and controls UI.
- **Feature Preservation**: Keep all existing movie player features (subtitles, audio/quality selector, speed settings, keyboard shortcuts, progress updates, and fullscreen mode) fully operational.
- **Optimization**: Optimize the video player for fast load times and a premium, ultra-smooth user experience.
- **General Rule**: Preserve all existing application functionality unless explicitly requested otherwise.
